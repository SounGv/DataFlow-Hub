## 4. สรุป Cleaning Rules ที่ Sync Engine ต้องทำ (Phase 2)
1. ตัดแถวหัวซ้ำ/แถวขยะ (case id ไม่ตรง `GV\d+` และไม่มีข้อมูลลูกค้า)
2. วันที่: parse หลาย format, ตรวจปี — ถ้า 1957–1970 หรือ >2500 ให้ปรับ พ.ศ.↔ค.ศ. (±543); ถ้าแก้ไม่ได้ → flag `needs_review`
3. เบอร์โทร: strip ช่องว่าง/ขีด, เติม 0 นำหน้าถ้า 9 หลัก, แปลง E.164
4. Shop: map ผ่านตาราง alias → shop canonical
5. Product: uppercase + trim → map เข้า products
6. กลุ่มปัญหา: map เข้า enum 5 ค่า + `other`
7. Dedupe case ด้วย `case_code` (เก็บแถวที่ข้อมูลสมบูรณ์กว่า, บันทึก conflict ลง audit)
8. Boolean cols (TRUE/FALSE) → boolean
9. ทุกแถว raw เก็บ `source_sheet`, `source_row`, `synced_at` เพื่อ traceability
